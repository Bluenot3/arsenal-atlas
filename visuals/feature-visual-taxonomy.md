# Feature Visual Taxonomy

Use consistent visual language so Arsenal feels like one platform.

## Visual Shapes

- Agent: worker card with role, model, tools, knowledge, channels, status.
- Business Space: command-center cockpit with ROI, readiness, proposal, outputs.
- Z-Engine: creation canvas with prompt, route, artifact preview, save/export.
- Vault: file tree plus proof cards and provenance.
- Flow: node path with trigger, action, approval, output, replay.
- Marketplace asset: product card with creator, install/purchase, category, proof.
- Admin control: dense operational table with risk, status, owner, last event.
- Credit/billing event: ledger row with source, amount, balance, user/tier, status.

## Status Color Model

- Ready / marketable: green.
- Needs polish / partial: amber.
- Risk / blocked: red.
- Admin / gated: violet.
- Unknown: gray.

## Feature Card Anatomy

```txt
┌────────────────────────────────────┐
│ Feature name            Status     │
│ One-line buyer value               │
│ Evidence: route/service/table      │
│ Revenue path | Risk | Screenshot   │
└────────────────────────────────────┘
```

## Data Cards To Use In Dashboards

- Routes by feature cluster.
- Features by status.
- Revenue priority by feature.
- Risk severity by area.
- AI routes by provider path.
- Credit-bearing surfaces.
- Screenshot readiness by asset type.
- Admin control coverage by risk level.
